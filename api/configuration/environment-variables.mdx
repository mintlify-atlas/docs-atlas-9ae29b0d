## Overview

The Hyperbolic AgentKit uses environment variables to configure API keys, feature flags, and system settings. Variables are loaded from a `.env` file in the project root.

## Setup

Copy the example environment file:

```bash
cp .env.example .env
```

Edit `.env` with your actual API keys and configuration.

## Required Variables

### Core APIs

<ParamField path="ANTHROPIC_API_KEY" type="string" required>
  Anthropic API key for Claude LLM. Get from [Anthropic Console](https://console.anthropic.com/dashboard).
</ParamField>

<ParamField path="CDP_API_KEY_NAME" type="string" required>
  Coinbase Developer Platform API key name. Get from [CDP Portal](https://portal.cdp.coinbase.com/access/api).
</ParamField>

<ParamField path="CDP_API_KEY_PRIVATE" type="string" required>
  Coinbase Developer Platform private key.
</ParamField>

<ParamField path="HYPERBOLIC_API_KEY" type="string" required>
  Hyperbolic API key for GPU compute tools. Get from [Hyperbolic Settings](https://app.hyperbolic.xyz/settings).
</ParamField>

### OpenAI (Required for Voice Agent)

<ParamField path="OPENAI_API_KEY" type="string">
  OpenAI API key. Required only if using the voice agent interface. Get from [OpenAI Portal](https://platform.openai.com/api-keys).
</ParamField>

## Configuration Variables

### SSH Configuration

<ParamField path="SSH_PRIVATE_KEY_PATH" type="string" default="~/.ssh/id_rsa">
  Path to SSH private key for GPU instance access. **Must be an RSA key** (ed25519 not supported).
  
  Generate an RSA key:
  ```bash
  ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
  ```
  
  Add public key to [Hyperbolic Settings](https://app.hyperbolic.xyz/settings).
</ParamField>

### Character Configuration

<ParamField path="CHARACTER_FILE" type="string" default="characters/default.json">
  Path to character configuration JSON file. Can be comma-separated for multiple characters.
</ParamField>

### Knowledge Base Configuration

<ParamField path="USE_TWITTER_KNOWLEDGE_BASE" type="boolean" default="false">
  Enable Twitter knowledge base for scraping KOL tweets. At least one knowledge base (Twitter or Podcast) must be enabled.
</ParamField>

<ParamField path="USE_PODCAST_KNOWLEDGE_BASE" type="boolean" default="true">
  Enable Podcast knowledge base for transcript processing.
</ParamField>

### Security Settings

<ParamField path="ALLOW_DANGEROUS_REQUEST" type="boolean" default="true">
  Allow HTTP request tools. **Must be true** to use request tools. Set to false in production for security.
</ParamField>

## Feature Flags

### Core Toolkits

<ParamField path="USE_COINBASE_TOOLS" type="boolean" default="true">
  Enable Coinbase CDP tools for blockchain operations.
</ParamField>

<ParamField path="USE_HYPERBOLIC_TOOLS" type="boolean" default="true">
  Enable Hyperbolic GPU compute tools.
</ParamField>

<ParamField path="USE_TWITTER_CORE" type="boolean" default="true">
  Enable core Twitter API integration.
</ParamField>

<ParamField path="USE_BROWSER_TOOLS" type="boolean" default="true">
  Enable browser automation tools (Playwright). **Recommended for maximum agent functionality.**
</ParamField>

### Twitter Tools

<ParamField path="USE_TWEET_REPLY_TRACKING" type="boolean" default="true">
  Enable tracking of replied tweets to avoid duplicates.
</ParamField>

<ParamField path="USE_TWEET_REPOST_TRACKING" type="boolean" default="true">
  Enable tracking of reposted tweets.

<ParamField path="USE_TWEET_DELETE" type="boolean" default="true">

</ParamField>

<ParamField path="USE_USER_ID_LOOKUP" type="boolean" default="true">
  Enable Twitter user ID lookup by username.

  Enable retrieving user's recent tweets.

<ParamField path="USE_RETWEET" type="boolean" default="true">

</ParamField>

### Additional Tools

<ParamField path="USE_DEPLOY_MULTITOKEN" type="boolean" default="true">
  Enable multi-token deployment tools.
</ParamField>

<ParamField path="USE_WEB_SEARCH" type="boolean" default="true">
  Enable web search via DuckDuckGo.
</ParamField>

<ParamField path="USE_REQUEST_TOOLS" type="boolean" default="true">

</ParamField>

## Example .env File

```bash

# OpenAI (Required for voice agent)
OPENAI_API_KEY=sk-...

# Twitter Integration (Optional)

TWITTER_ACCESS_TOKEN_SECRET=...
TWITTER_BEARER_TOKEN=...
TWITTER_CLIENT_ID=...

# Character Configuration

# Knowledge Base Configuration
USE_TWITTER_KNOWLEDGE_BASE=false
USE_PODCAST_KNOWLEDGE_BASE=true

# Security Settings
ALLOW_DANGEROUS_REQUEST=true

# Core Toolkits
USE_COINBASE_TOOLS=true
USE_HYPERBOLIC_TOOLS=true
USE_TWITTER_CORE=true
USE_BROWSER_TOOLS=true

# Twitter Tools
USE_TWEET_REPLY_TRACKING=true
USE_TWEET_REPOST_TRACKING=true
USE_TWEET_DELETE=true
USE_USER_ID_LOOKUP=true
USE_USER_TWEETS_LOOKUP=true
USE_RETWEET=true

# Additional Tools
USE_DEPLOY_MULTITOKEN=true
USE_WEB_SEARCH=true
USE_REQUEST_TOOLS=true
USE_WRITING_AGENT=true
```

## Security Best Practices

1. **Never commit `.env` to version control**
   - Add `.env` to `.gitignore`
   - Use `.env.example` as a template (without actual keys)

2. **Restrict file permissions**
   ```bash
   chmod 600 .env
   ```

3. **Use different keys for development and production**

4. **Rotate API keys regularly**

5. **Set `ALLOW_DANGEROUS_REQUEST=false` in production** if request tools aren't needed
